TCP 握手是三次，挥手是四次为什么？

三次握手：防止历史连接、同步序列号、避免资源浪费。四次挥手：TCP 全双工，双方需各自关闭；FIN 和 ACK 分开发送。

 计算机网络

TCP 三次握手和四次挥手的区别，为什么少一次？

握手时服务端 SYN+ACK 可合并；挥手时被动方可能还有数据要发，FIN 不能与 ACK 同时发。

同步阻塞、同步非阻塞、异步阻塞、异步非阻塞的区别

同步/异步：调用方是否等待结果。阻塞/非阻塞：等待时线程是否挂起。四种组合的典型场景与 I/O 模型。

 并发

乐观锁和悲观锁的区别

悲观锁：先加锁再操作（synchronized、行锁）。乐观锁：无锁，通过版本号/CAS 检测冲突后重试。

腾讯二面

详细笔记

点击上方卡片可跳转，以下为简要提纲，可自行补充完善。

TCP 三次握手

1. 客户端 SYN → 2. 服务端 SYN+ACK → 3. 客户端 ACK。为什么三次：防止已失效连接请求、同步初始序列号。

握手 vs 挥手

挥手多一次：被动方收到 FIN 后可能还有数据要发，先 ACK，发完再 FIN。

四种 I/O 模型

同步阻塞(BIO)、同步非阻塞(NIO 轮询)、异步阻塞(少见)、异步非阻塞(AIO/NIO 多路复用)。

乐观锁 vs 悲观锁

悲观：synchronized、SELECT FOR UPDATE。乐观：版本号、CAS。高并发读多写少用乐观。

Redis 分布式锁要点

SET key value NX EX seconds；value 用 UUID 防误删；Lua 保证原子性；Redisson 看门狗续期。

秒杀系统要点

分层限流、Redis 预减、MQ 削峰、数据库唯一索引防超卖、分库分表、缓存预热。

a++ vs ++a

i++ 需保存旧值再 +1；++i 直接 +1。单条语句中 JIT 优化后差异可忽略。

数组 vs 链表（内存视角）

数组：O(1) 寻址、缓存命中高。链表：每节点额外 8/16 字节指针、内存碎片。内存紧张选数组。

Embedding 应用

语义检索、推荐、聚类、分类。向量维度通常 256～1536。相似度：余弦、欧氏。

RAG 流程

文档分块 → Embedding → 存入向量库 → 查询时检索 TopK → 拼入 Prompt → LLM 生成。

构造 n 个数差值为 k

如 [0, k, 2k, ...] 或 [k, -k, k, -k, ...]。注意边界与 n、k 关系。

二叉树路径和

DFS(root, target - root.val)；到叶子且 target==0 则找到；回溯时恢复 path。

计算机基础总结

计网：TCP/UDP、HTTP/HTTPS、DNS。OS：进程线程、虚拟内存、调度。数据库：索引、事务、锁。
