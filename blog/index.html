TCP 握手是三次，挥手是四次为什么？

计算机网络

三次握手：

  • 防止历史连接：若用两次，旧 SYN 可能迟到，服务端误开连接
  • 同步序列号：双方确认对方的初始序列号
  • 避免资源浪费：两次握手时服务端无法确认客户端是否收到 SYN+ACK

四次挥手：

  • TCP 全双工，双方需各自关闭
  • 被动方收到 FIN 后可能还有数据要发，先 ACK，发完再发 FIN
  • FIN 和 ACK 不能合并

TCP 三次握手和四次挥手的区别，为什么少一次？

计算机网络

握手时服务端 SYN+ACK 可合并；挥手时被动方可能还有数据要发，FIN 不能与 ACK 同时发，所以多一次。

同步阻塞、同步非阻塞、异步阻塞、异步非阻塞的区别

操作系统 / 并发

同步/异步：调用方是否等待结果返回。

阻塞/非阻塞：等待时线程是否挂起。

  • 同步阻塞 (BIO)：调用后一直等，线程阻塞
  • 同步非阻塞 (NIO 轮询)：立即返回，轮询检查是否就绪
  • 异步阻塞：少见
  • 异步非阻塞 (AIO / NIO 多路复用)：注册回调，就绪后通知

乐观锁和悲观锁的区别

并发

悲观锁：先加锁再操作。如 synchronized、SELECT FOR UPDATE、行锁。

乐观锁：无锁，通过版本号/CAS 检测冲突后重试。

读多写少、冲突少时用乐观锁；写多、冲突多时用悲观锁。

a++ 和 ++a 谁性能高？

Java 某大厂二面

i++ 需保存旧值再 +1；++i 直接 +1。现代 JVM 下字节码层面几乎无差异，JIT 会优化。单条语句中差异可忽略。

在内存有限的机器上尽量使用数组还是链表？

Java / 数据结构 某大厂二面

数组：连续内存、缓存友好、无额外指针开销、O(1) 寻址。

链表：节点分散、每节点 8/16 字节指针、缓存不友好、内存碎片。

内存紧张时数组更优。
